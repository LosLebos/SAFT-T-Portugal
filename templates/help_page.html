Welcome to SAF-T Tools

This application is designed to assist with the generation and management of SAF-T (Standard Audit File for Tax) data, specifically tailored for the Portuguese SAF-T PT format (Version 1.04_01). It allows users to manage master data, upload transactional data (upcoming feature), and generate compliant XML files.

Whether you are an authenticated user managing your own company's data or exploring the application in Demo Mode, we aim to provide a straightforward experience.

Current Features

  • User Authentication: Secure login and registration for managing your own data.
  • Demo Mode: Unauthenticated users can explore the application using pre-populated sample data. Data viewed or generated in demo mode is for illustrative purposes only.
  • Customer Data Management:
    • View a list of customers (filtered by user in authenticated mode, demo data otherwise).
    • (Future: Upload, Add, Edit, Delete Customer records).
  • SAF-T XML Generation:
    • Generate a SAF-T PT XML file based on the current user's data (or demo data).
    • The generated XML includes Header, MasterFiles (Customers, Products, Suppliers, GeneralLedgerAccounts), and placeholder GeneralLedgerEntries.
    • XML is validated against the official SAFTPT1_04_01.xsd schema before download.
    • Error reporting if the generated XML fails schema validation.
  • Master Data Management (Backend):
    • Functionality to manage Suppliers, Products, and General Ledger Accounts is supported by the backend models and database structure, though UI for direct management is pending.
    • Demo data includes samples for these master files.

Upcoming Features / Roadmap

  • Full Master Data Management UI: Web interfaces for adding, editing, and deleting Customers, Suppliers, Products, and General Ledger Accounts.
  • Transactional Data Upload: Functionality to upload GeneralLedgerEntries (Journal, Transactions, Lines) from CSV or other formats.
  • SourceDocuments Data Upload: Support for SalesInvoices, MovementOfGoods, WorkingDocuments, and Payments.
  • Enhanced Header Configuration: UI to manage and customize Header data for SAF-T generation.
  • User Profile Management: Allow users to manage their profile information.
  • Data Validation Enhancements: More granular validation during data input and upload.
  • Comprehensive Error Reporting: Improved user feedback for all operations.
  • User Roles and Permissions: (If applicable) Differentiating user capabilities.
